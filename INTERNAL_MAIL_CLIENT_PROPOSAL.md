## أولاً: نبذة عن الوحدة

وحدة بريد داخلي متكاملة مع النظام الحالي تتيح لجميع الموظفين التواصل فيما بينهم ضمن المنصة مباشرة، مع إمكانية إرسال مرفقات وتعميمات إدارية وإشعارات فورية، بما يضمن سرية المراسلات وتوثيقها وربطها بالهيكل الإداري للمؤسسة.

---

## ثانياً: المميزات التفصيلية

### 1. الرسائل المباشرة

| الميزة | الوصف |
|--------|-------|
| إرسال رسالة | إرسال رسالة مع عنوان ونص إلى أي موظف في النظام |
| CC (نسخة كربونية) | إمكانية إضافة موظفين آخرين للاطلاع على الرسالة (مدير، مسؤول، زميل) |
| الرد والرد المتسلسل | الرد على رسالة مع الحفاظ على سلسلة المحادثة كاملة |
| صندوق الوارد | عرض جميع الرسائل الواردة مع ترقيم الصفحات وإمكانية الفرز |
| صندوق الصادر | عرض جميع الرسائل المُرسلة |
| تحديد كمقروءة | تحديد رسالة أو جميع الرسائل كمقروءة |
| حذف الرسائل | حذف الرسالة من صندوق الموظف مع بقائها عند الطرف الآخر |
| عداد غير المقروءة | عرض عدد الرسائل غير المقروءة بشكل دائم |
| البحث | البحث في الرسائل بالعنوان أو اسم المرسل أو المحتوى |

### 2. المرفقات

| الميزة | الوصف |
|--------|-------|
| إرفاق ملفات | إرفاق ملفات مع الرسالة (PDF، صور، مستندات Office) |
| مرفقات متعددة | إمكانية إرفاق أكثر من ملف في رسالة واحدة |
| تحميل المرفقات | تحميل الملفات المرفقة مباشرة من الرسالة |
| حد الحجم | حد أقصى قابل للتعديل لحجم الملف الواحد (افتراضي: 10 ميغابايت) |
| الصيغ المدعومة | PDF, JPG, PNG, DOC, DOCX, XLS, XLSX |

### 3. التعميمات الإدارية

| الميزة | الوصف |
|--------|-------|
| إنشاء تعميم | إرسال رسالة رسمية من الإدارة لجميع الموظفين |
| تعميم لقسم | إرسال تعميم لقسم محدد فقط |
| تثبيت التعميم | تثبيت تعميم مهم في أعلى القائمة |
| تأكيد القراءة | تتبع من قرأ التعميم ومن لم يقرأه مع نسبة الاطلاع |
| أرشيف التعميمات | أرشفة التعميمات القديمة مع إمكانية الرجوع إليها |

### 4. الإشعارات

| الميزة | الوصف |
|--------|-------|
| إشعار فوري | إشعار لحظي عند وصول رسالة جديدة (داخل المنصة) |
| إشعار التعميم | إشعار خاص عند صدور تعميم إداري |
| جرس الإشعارات | أيقونة إشعارات مع عداد بالرسائل والتعميمات الجديدة |
| قائمة الإشعارات | عرض جميع الإشعارات مع إمكانية تحديدها كمقروءة |

### 5. الصلاحيات والأمان

| الميزة | الوصف |
|--------|-------|
| صلاحيات الإرسال | التحكم بمن يحق له إرسال الرسائل |
| صلاحيات التعميمات | فقط المخوّلون (مدراء، HR) يمكنهم إنشاء تعميمات |
| خصوصية الرسائل | كل موظف يرى رسائله فقط |
| سجل مراجعة | توثيق جميع العمليات (إرسال، قراءة، حذف) لأغراض المراجعة |

### 6. التكامل مع النظام

| الميزة | الوصف |
|--------|-------|
| دليل الموظفين | اختيار المستلم و CC من قائمة الموظفين المسجلين بالنظام |
| الأقسام | تصفية الموظفين حسب القسم عند الإرسال |
| الهيكل الإداري | عرض المدير المباشر لتسهيل إضافته كـ CC |
| المصادقة الموحدة | نفس حساب الدخول للنظام بدون تسجيل إضافي |

## ثالثاً: المخرجات

| المخرج | الوصف |
|--------|-------|
| Backend API | واجهة برمجية كاملة وموثقة لجميع عمليات البريد الداخلي |
| قاعدة البيانات | جداول البريد والمرفقات والإشعارات مع العلاقات |
| التكامل | ربط كامل مع نظام المستخدمين والصلاحيات والبوابة |
| التوثيق | توثيق فني للـ API مع أمثلة الطلبات والاستجابات |
| بيئة Docker | جاهز للنشر ضمن بيئة Docker الحالية |

## رابعاً: خارج نطاق العمل

| البند | ملاحظة |
|-------|--------|
| واجهة المستخدم (Frontend) | العرض يشمل الباك اند فقط، الفرونت اند بعرض منفصل |
| تطبيق الموبايل | غير مشمول |
| إرسال إيميل خارجي | غير مشمول (يمكن إضافته لاحقاً) |
| محادثات فورية (Chat) | غير مشمول - هذا نظام بريد وليس دردشة |

## خامساً: الجدول الزمني

| اليوم | المهمة |
|-------|--------|
| **1** | إعداد بنية السيرفس وقاعدة البيانات والربط بالنظام |
| **2 - 3** | تطوير نظام الرسائل (إرسال، استقبال، رد، CC، سلاسل المحادثة) |
| **4** | نظام البحث والفلترة وصناديق البريد |
| **5 - 6** | نظام المرفقات (رفع، تخزين، تحميل، التحقق) |
| **7** | التعميمات الإدارية (إنشاء، إرسال، تتبع القراءة) |
| **8** | الإشعارات الفورية (SSE) وجرس الإشعارات |
| **9** | الصلاحيات والأمان والربط النهائي مع Gateway |
| **10** | الاختبار الشامل والتوثيق والتسليم |
